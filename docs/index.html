Warum RoboScope?

Die Lücke zwischen Robot Framework und dem Team schließen

Wer schon einmal versucht hat, ein Team ohne tiefe Kommandozeilen-Erfahrung für Robot Framework zu begeistern, kennt das Muster: Die Syntax ist verständlich, die Keyword-Idee leuchtet ein — aber dann kommt das Setup. Git klonen, virtuelle Umgebung anlegen, Abhängigkeiten installieren, Tests über das Terminal starten, Output-Dateien suchen und öffnen.

Spätestens hier steigen die Kolleg:innen aus, die eigentlich die besten Testideen hätten.

Robot Framework bringt eigentlich alles mit, um auch von weniger technisch versierten Nutzer:innen eingesetzt zu werden. Die Keyword-Driven-Architektur ist dafür wie geschaffen. Aber das Tooling drum herum setzt eben doch voraus, dass man sich im Terminal wohlfühlt.

RoboScope schließt genau diese Lücke: Eine Web-Oberfläche, über die das gesamte Team — von Fachtester:innen bis Product Owner — Tests verwalten, ausführen und auswerten kann, ohne das Terminal zu öffnen.

In Minuten startklar

Kein Cloud-Account nötig. Alles läuft lokal auf deinem Rechner.

ZIP Download (Offline)

Kein Git, kein Node.js, kein Internet nötig. Nur Python 3.10+ muss installiert sein.

1

Download & Entpacken

ZIP von GitHub Releases herunterladen und entpacken.

2

Installieren

chmod +x install-mac-and-linux.sh ./install-mac-and-linux.sh
install-windows.bat
3

Starten

./start-mac-and-linux.sh # http://localhost:8145
start-windows.bat REM http://localhost:8145

Docker Compose

PostgreSQL + Nginx, production-ready. Benötigt nur Docker und Docker Compose.

1

Repository klonen

git clone https://github.com/viadee/roboscope.git cd roboscope
2

Production starten

docker compose up -d # http://localhost:8000
3

Oder Dev-Modus (SQLite)

docker compose -f docker-compose.dev.yml up # http://localhost:5173

Git Clone (Dev-Setup)

Quellcode bearbeiten und beitragen. Benötigt Python 3.12+, Node.js 20+ und uv.

1

Voraussetzungen

# uv installieren curl -LsSf https://astral.sh/uv/install.sh | sh
2

Clone & Install

git clone https://github.com/viadee/roboscope.git cd roboscope cp backend/.env.example backend/.env make install
3

Dev-Server starten

make dev # Backend: http://localhost:8000 # Frontend: http://localhost:5173

Default-Login bei allen Varianten: admin@roboscope.local / admin123
